POST SHA-256 Signature Deposit Withdrawal Callback Support

Overview

ABC FINPAY provides deposit, withdrawal, balance checking, and status checking APIs. Requests use SHA-256 signatures built from concatenated field values using your merchant key and merchant value.

Merchant Credentials

Merchant Key and Merchant Value are required for signature generation.

Service Scope

  • Deposit transaction creation
  • Withdrawal transaction creation
  • Balance checking
  • Status checking
  • Callback update notifications

Authentication & Signature

All endpoints use a SHA-256 signature generated from concatenated values in the required order. Do not add separators unless the documentation explicitly says so. 

Signature = SHA256(concatenated_string)

Signature Code Examples

C#

public static class SHA_256
{
    public static string GenerateSHA256String(string inputString)
    {
        SHA256 SHA256 = SHA256Managed.Create();
        byte[] bytes = Encoding.UTF8.GetBytes(inputString);
        byte[] hash = SHA256.ComputeHash(bytes);
        return GetStringFromHash(hash);
    }
    private static string GetStringFromHash(byte[] hash)
    {
        StringBuilder result = new StringBuilder();
        for (int i = 0; i < hash.Length; i++)
        {
            result.Append(hash[i].ToString("X2"));
        }
        return result.ToString().ToLower();
    }
}

PHP

<?php
function signature($source)
{
    return hash('sha256', $source);
}
?>
